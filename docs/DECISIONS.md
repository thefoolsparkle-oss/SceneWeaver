# SceneWeaver 重要技术决策

## 决策 1：使用 Tauri v2 + React + TypeScript + Vite

- **理由**：V2 规范推荐 Tauri；Tauri 生成的小型 Windows 安装包更符合普通用户预期；React + TS + Vite 是成熟的前端组合。
- **时间**：2026-07-18
- **负责人**：OpenCode

## 决策 2：Rust 使用 GNU 工具链 + MinGW-w64

- **理由**：开发环境缺少 MSVC Build Tools（Visual Studio），用户明确不希望安装 Visual Studio。MinGW-w64 可提供完整链接器，避免强制依赖 VS。
- **风险**：部分 Windows 原生 API 或 `-sys` crate 可能需要额外适配。
- **缓解**：优先使用跨平台 crate，针对 Tauri Windows 插件做单独验证。
- **状态**：已移除全局 `--exclude-all-symbols`，因为它会破坏 Rust 测试二进制。当前 GNU 工具链的桌面链接兼容性仍待单独处理；标准测试不依赖该参数。
- **环境**：本机使用 MSYS2 MinGW-w64；执行 Cargo 前需将 `C:\msys64\mingw64\bin` 加入当前 shell 的 `PATH`，使 `gcc` 与 `windres` 可被 Tauri 构建脚本发现。
- **时间**：2026-07-18

## 决策 3：SQLite 作为业务数据库

- **理由**：V2 规范明确要求消费者版使用 SQLite；无需独立数据库服务；WAL 模式支持并发读写。
- **时间**：2026-07-18

## 决策 4：FFmpeg / ffprobe 作为可选外部依赖

- **理由**：FFmpeg 体积大、许可证复杂，完全打包会显著增加安装包体积。首发版本优先检测系统 PATH，未检测到时引导用户下载或自动下载静态构建。
- **后续**：稳定后可考虑打包轻量侧车或调用 Windows Media Foundation。
- **时间**：2026-07-18

## 决策 5：索引分层（Level 0-4）

- **理由**：大型素材库需要尽快可用。Level 2（缩略图 + 代表帧 + 视觉向量）完成后即可搜索，增强分析异步进行。
- **时间**：2026-07-18

## 决策 6：云端能力默认关闭

- **理由**：隐私优先；无 API Key 用户也能使用核心流程。
- **时间**：2026-07-18

## 决策 7：场景包作为插件式扩展

- **理由**：保持通用核心简洁，ACG 作为首发包，不硬编码到数据模型。
- **时间**：2026-07-18

## 决策 8：资产以“素材库 + 规范化路径”唯一

- **理由**：同一个本地文件可被多个用户授权的素材库引用；全局路径唯一会导致后扫描的素材库覆盖资产归属。
- **迁移**：数据库升级到 schema v3，并记录每次扫描的 `last_seen_scan_at`，以支持不加载整库路径列表的离线文件识别。
- **时间**：2026-07-18

## 决策 9：原文件操作仅接受资产 ID

- **理由**：避免渲染层通过宽泛 Shell 权限打开任意文件或 URL；后端从本地数据库解析已授权资产路径。
- **时间**：2026-07-18
