### 当前可执行核心 smoke 测试

`cargo run --manifest-path src-tauri/Cargo.toml --bin core_smoke` 在系统临时目录运行并清理测试数据，已覆盖：

- 中文且带空格的素材库路径；
- 超过 260 字符的中文 Windows 素材路径；
- 图片扫描、SQLite 写入与真实 JPEG 缩略图；
- 第二次扫描的不变文件跳过；
- 修改同一文件后只重新处理该素材并更新元数据；
- 删除文件后的离线状态更新。
- 损坏 PNG 不会中断同一素材库中其它文件的扫描。
- 默认与自定义选片集合持久化、上下/拖拽排序，以及真实 CSV、JSON、EDL、FCPXML 写出。
- 默认与自定义选片集合 PNG 联系表写出；片段项应优先使用关键帧。
- 片段写入、幂等加入默认选片、片段时间范围回读，以及重新切分时清理失效片段选片项。
- 片段范围写入 CSV、JSON、EDL、FCPXML；推荐入/出点应优先于片段范围。
- FCPXML 中的中文 Windows 文件 URI 编码与图片媒体属性。
- FCPXML 的格式/素材资源 ID 不重复，且 23.976、29.97、59.94 与整数帧率生成正确的帧时长。
- Must / Should / Must Not 条件解析，以及编辑其中一项时只修改目标条件。
- 图片视觉特征持久化，以及按参考资产召回相似图片。
- 单一素材并存多个 Provider 向量，且查询按 Provider 隔离。
- 已索引源文件被修改后，旧 Provider 向量、镜头片段、片段 Selects 与实体素材反馈向量会被失效或同步，避免使用替换前内容。
- 既有 v6 单向量数据库升级为多 Provider 索引时保留旧向量。
- 新建数据库的 schema 版本为 v8，重开不会重放 v7→v8 迁移；Entity 颜色参考只与颜色 Provider 比较。
- 语义模型未安装或配对缓存文件缺失时明确降级、绝不隐式下载；随安装器的 ONNX Runtime 资源存在。
- 已知中文 ACG/镜头词会以可审计的本地词典展开为 CLIP 英文提示；未知中文不应伪造语义向量召回。
- 含 Must Not 的中文请求只将 Must/Should 词送入 CLIP 提示；被排除的字幕、UI 等词不得反向参与语义召回。
- 实体正参考图写入、按实体召回，以及按实体归属移除错误参考图。
- Entity 参考向量按 Provider 隔离；v7→v8 迁移保留既有颜色参考，新语义 Provider 可独立参与实体召回。
- 文本查询中的 Entity 名称或别名会合并本地参考图候选；仅处于 Must Not 的 Entity 名称不得触发正向实体召回。
- 用户确认的 ACG 素材标签持久化、标签参与 Must 条件搜索，以及更正时安全移除标签。
- ACG 标签命中会从未满足的 Should 中移除并显示标签原因；关键词、CLIP 与 Entity 候选合并后按稳定的解释分数排序。
- 同一分数层内的候选按素材库稳定轮换，保留每个素材库的原相对顺序；不同分数层不得因多样性调整而互相跨越。
- ACG 查询预设和同义词展开；游戏 UI 等 Must Not 同义词必须保持硬排除。
- 暂停扫描任务的检查点写入，并在重新打开 SQLite 后读取。
- 暂停任务在恢复队列中保持暂停，不会在应用重启后被自动重新执行。
- 应用中断前处于 `running` 的任务在重开 SQLite 后应保留 checkpoint 并以 `pending` 状态进入队列；同一批中的 `paused` 任务不得被改写或自动执行。
- 若 FFmpeg/ffprobe 可用，合成红蓝切换 MP4 的元数据、镜头、关键帧、短预览、黑帧/模糊/质量指标与视频代表帧索引。
- 视频首次扫描自动持久化镜头片段及派生结果；FFmpeg 缺失或派生失败时不得中断素材入库。
- 代表关键帧会写入保守的 `subtitle_present` 提示；“字幕 / subtitle”必须能作为 Must 命中、Should 加分和 Must Not 排除视频，且不得宣称为 OCR。
- 片段的黑帧比例 ≥ 85% 与模糊分 ≥ 80% 应分别识别为“黑帧 / 黑屏”和“模糊 / blur”条件，支持 Must 命中、Should 加分和 Must Not 排除；解释须说明它们是代表帧启发式。
- 双下角同时存在小面积高对比 HUD 图形的合成视频应自动写入 `game_ui=true`；`UI` / `HUD` 支持 Must 命中与 Must Not 保留无 HUD 镜头。用户确认的中文“游戏 UI”标签不得被自动 HUD 条件覆盖。
- 多个结构化片段 Must/Must Not 必须由同一个镜头同时满足；含有字幕镜头和无字幕镜头的同一视频，在“不要字幕”时应保留，反之“字幕且不要模糊”不得由不同镜头拼接命中。
- 镜头检测和关键帧/短预览派生均有 90 秒超时；超时后应终止并回收 FFmpeg 子进程，再将该素材安全降级而非阻塞扫描。

它已覆盖可选的合成 FFmpeg 视频派生、基础搜索条件和暂停状态恢复；真实用户视频、完整任务暂停/恢复交互、GUI 搜索流程和专业剪辑软件实际导入仍需补充为独立集成用例。

### 明确触发的语义模型 smoke

`SCENEWEAVER_DOWNLOAD_SEMANTIC_MODEL=1 cargo run --manifest-path src-tauri/Cargo.toml --bin semantic_model_smoke` 仅在环境变量明确确认时从模型源下载配对 CLIP 编码器到独立临时目录，实际生成 512 维图像与文本向量、写入 SQLite 后验证文本检索和 Entity 语义参考检索，最后清理临时模型与数据库。它不属于默认测试，确保普通用户和 CI 不会被隐式下载权重。

`.github/workflows/ci.yml` 在 Windows GitHub Runner 对 `main` 推送和 Pull Request 执行前端 lint/test/build、Rust 格式检查、Rust 测试编译和默认 `core_smoke`。语义模型下载 smoke、FFmpeg 派生断言和安装器交互 smoke 仍保留给显式本地验证。

CI 必须从干净工作目录通过：`build.rs` 会在 Tauri 读取 `bundle.resources` 前，从 Cargo.lock 锁定的 `webview2-com-sys` x64 依赖暂存 `WebView2Loader.dll`。这覆盖了“本机旧 target 文件掩盖 DLL 缺失”的回归场景；2026-07-19 的 Windows GitHub Runner 已实际通过该编译路径与默认 core smoke。本地 NSIS smoke 继续验证安装后的 DLL 与 EXE 同级且可启动。

2026-07-19 已实际运行该 smoke：图像与文本各返回 512 维向量，图文余弦值为 0.279；SQLite 文本检索和 Entity 语义参考检索均返回目标素材，临时模型目录已由 smoke 清理。
