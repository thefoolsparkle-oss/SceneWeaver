# SceneWeaver 实施进度

> 本文档实时记录各阶段完成情况、测试结果与下一轮任务。
> 所有记录基于 `SceneWeaver_PRODUCT_REDESIGN.md`（V2 规范）。

## 当前阶段

阶段 1：本地素材库与增量扫描（纵向闭环实现中；质量门禁已开始建立）

阶段 2：已实现扫描时自动执行的 FFmpeg 场景边界切分、关键帧、短预览、基础黑帧/模糊检测与片段持久化；片段可展示质量指标、按需播放短预览并加入 Selects。搜索结果的视频卡片会显示已定位片段数；打开片段面板时只展示本次结构化条件命中的镜头，完成搜索到精确片段、时间码和片段级选片的闭环。关键帧的下部高对比度文字形态会以保守启发式写入 `subtitle_present`；“字幕 / subtitle”“黑帧 / 黑屏”“模糊 / blur”现均可作为视频素材的 Must / Should / Must Not 条件。黑帧比例达到 85%、模糊分达到 80% 才会产生对应提示；它们和字幕提示都属于代表帧启发式，不等同 OCR 或真实画质评测。结构化 Must/Must Not 会在同一片段上组合：包含多个不同镜头的视频不会因其中一个不合格镜头而被整条排除，但“字幕且不要模糊”等组合也不会由两个不同镜头错误拼接满足。FFmpeg 缺失、镜头检测失败或单个派生文件失败均不会中断素材入库，视频仍可搜索并可稍后手动重试切分。镜头检测与派生各自的 90 秒超时会终止并回收 FFmpeg 子进程，避免遗留后台进程。2026-07-19 已用本机 FFmpeg 8.1.2 合成红蓝切换 MP4 实测自动扫描路径，验证片段、关键帧、短预览、黑帧/模糊/质量指标和字幕字段均写入 SQLite 与缓存；smoke 同时验证字幕、黑帧和模糊条件的 Must 命中、Must Not 安全片段保留及同镜头组合约束。

阶段 3：已实现本地持久化图片视觉特征与相似图片召回，并接入可选的本地 CLIP ViT-B/32 图文 Provider。设置页只有在用户点击后才下载模型并允许重建语义索引；运行库随安装器携带，模型缺失、运行库缺失或推理失败均降级为关键词/颜色检索。CLIP 模型的中文通用语义质量尚未验证，不将其宣称为中文模型。

为提高常见中文创作搜索的可用性，已加入透明的本地中英提示词典：角色、雨夜、侧脸、回头、战斗、字幕、近/远景等已知词会扩展为 CLIP 英文提示；未知中文专有名词保持关键词降级。此机制不替代通用中文模型。

阶段 3 索引基础：`asset_embeddings` 已迁移为 `(asset_id, provider_id)` 复合主键；基础颜色特征与后续语义模型能够同时持久化，检索按 Provider 隔离，不会因模型切换覆盖旧索引。

`core_smoke` 已用合成 v6 SQLite 数据库验证上述迁移会保留既有向量，并允许同一素材写入第二个 Provider。

本轮还修复了 Entity 参考向量在多 Provider 数据库中未限定 Provider 的问题：实体颜色参考和资产颜色索引现在明确使用 `local-color-histogram`，不会把 24 维颜色向量与 CLIP 向量混合排名；smoke 同时验证新建数据库初始即为 v8，重开不会重放旧迁移。

本轮已从 ONNX Runtime 官方 Windows x64 CPU 发布包随应用加入 `onnxruntime.dll`；`core_smoke` 验证运行库存在且不会隐式下载模型。NSIS 资源配置会将其与 EXE 同级携带。

`semantic_model_smoke` 已在独立临时目录真实下载配对 CLIP 图像/文本编码器，完成 512 维图像与文本推理、SQLite 语义文本召回、实体语义参考召回并清理临时模型；这验证了动态 ONNX Runtime、模型缓存布局、Provider 的实际推理路径以及按 Provider 持久化检索。它仍不是中文语义效果评估。

阶段 4：已实现 Must / Should / Must Not 本地条件过滤、Should 软排序、结果解释、搜索历史，以及可直接编辑或移除的条件芯片。ACG 标签命中会在结果解释与 Should 完成状态中正确反映；关键词、CLIP 和 Entity 多路候选合并后会按可解释分数、更新时间和稳定 ID 排序。同分候选再按素材库稳定轮换，避免单个库挤占前排，并在结果原因中说明。视觉/元数据条件和跨 Provider 分数校准尚未完成。

语义 Provider 已安装时，搜索会将关键词与 CLIP 图文候选去重合并；Entity 候选也会进入同一排序。媒体类型、片段质量与 Must Not 仍由 SQLite 作为硬约束。结果解释会区分 CLIP 语义向量、颜色特征与 ACG 标签；同分候选按素材库轮换，当前尚未做跨路分数校准。

阶段 6：选片集合、移除/归档、评分、备注、推荐入/出点、拖拽排序，以及默认/自定义集合的 CSV/JSON/EDL/FCPXML/PNG 联系表导出已实现；片段范围与推荐入/出点会写入各时间线格式。剪辑软件导入验证尚未完成。

阶段 5：已实现通用 Entity 创建、名称/别名匹配、正/负本地参考图召回及误加参考图移除。实体参考向量现可按 Provider 持久化：颜色参考始终走颜色索引，安装并重建 CLIP 后参考图与素材反馈也会写入语义 Provider 并参与合并召回。视频代表帧会以“双下角同时出现小面积高对比元素”的严格启发式写入 `game_ui`；`UI` / `HUD` 可作为片段级 Must / Should / Must Not 条件，但它不是通用 UI 或菜单识别。实体置信度、真实中文语义效果评估与完整 ACG Pack 尚未完成。

实体现在既可从主搜索页直接选择，也可在自然语言请求中通过名称或别名自动识别并召回；名称/别名命中、颜色参考和（已重建时）语义参考会去重合并，卡片会显示命中的实体名称。仅出现在 Must Not 的实体不会被用作正向召回。无语义模型时仍可使用名称/别名和颜色参考，不会阻塞核心搜索。

ACG Creator Pack：已支持用户为素材添加本地确认标签，标签会持久化并参与 Must / Should / Must Not 组合查询；保守 HUD 提示可辅助简短 `UI` / `HUD` 片段筛选，中文“游戏 UI”仍保留给用户确认标签。自动战斗、视角与表情识别及通用 UI 识别仍待视觉模型能力完成。

ACG 查询预设已接入搜索页（角色近景、雨夜侧脸、战斗无 UI/字幕、剧情过场、情绪结尾）；预设先写入可编辑条件芯片。ACG 同义词对 Must Not 也作为硬排除写入 SQLite，而非错误地作为 Should 加分。

导出：FCPXML 的 Windows 源路径现以 UTF-8 百分号编码为 `file:///` URI，并依据素材类型输出音频或视频属性，减少中文与保留字符路径的导入歧义。每个格式和素材资源使用独立 ID；时间线采用首个有效帧率，视频素材资源保留已探测的帧率与分辨率（包括 NTSC 分数帧时长）。真实剪辑软件导入仍待验证。
