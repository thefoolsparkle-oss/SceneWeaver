# SceneWeaver V2 产品重构与实施规范

> 新中文定位：面向内容创作者的本地多模态素材搜索、整理与选片工作台  
> 新英文定位：Local-first Multimodal Media Search & Selects Workspace for Creators  
> 产品口号：说出你要的镜头，直接回到创作。  
> 文档性质：产品战略、MVP 边界、交互、技术架构、商业化与实施路线的统一规范

# 0. 执行摘要

SceneWeaver V2 不再试图成为一个简化版 Premiere、达芬奇或剪映，也不把“AI 自动粗剪”作为首发卖点。

产品首先解决一个高频、清晰、可量化的问题：

> 当创作者拥有大量分散的图片和视频素材时，能够通过自然语言、参考图和组合条件，在数秒内定位到具体素材和准确时间段，并立即回到原有剪辑流程。

V2 采用“通用核心 + 场景能力包”的策略：

- 通用核心服务所有视频创作者；
- ACG / 动漫游戏二创作为首个重点场景包；
- 后续可扩展 Vlog、短视频、电商、摄影、访谈、课程和企业素材场景；
- 不因追求广泛适用而牺牲首发产品的辨识度。

第一版的成功标准不是功能数量，而是用户是否完成以下闭环：

```text
选择素材文件夹 → 自动增量索引 → 输入自然语言 → 找到可用镜头 → 预览 → 收藏或导出 → 回到剪辑软件
```

# 1. 产品战略裁决

## 1.1 保留的核心价值

- 本地素材不搬家；
- 原视频默认不上传；
- 镜头级时间定位；
- 自然语言搜索；
- 结构化条件过滤；
- 参考图搜索；
- 可解释匹配；
- 素材收藏与选片；
- 与现有专业剪辑软件衔接。

## 1.2 删除的错误方向

首发版本不做：

- 完整时间线编辑器；
- 复杂入点、出点和转场编辑；
- 自动完成整条视频；
- 默认全量云端视觉分析；
- 强制 Docker、PostgreSQL、Redis；
- 企业级多租户和权限系统；
- 自动抓取网络影视资源；
- 内置版权内容数据库；
- 把视觉模型当作角色百科全书；
- 试图一次覆盖全部创作者类型。

## 1.3 产品北极星

> 每周被用户真正保留、打开或导出的搜索结果数量。

这比搜索次数更重要，因为它代表 SceneWeaver 找到的素材真正进入了创作流程。

# 2. 新产品定位

## 2.1 一句话定位

SceneWeaver 是一款本地优先的图片与视频素材智能工作台，帮助创作者用自然语言、参考图和组合条件快速找到素材，并将选中的片段送回现有剪辑软件。

## 2.2 产品不是

- 不是云盘；
- 不是素材交易平台；
- 不是完整剪辑器；
- 不是生成视频工具；
- 不是聊天机器人；
- 不是只服务 ACG 的封闭工具；
- 不是要求用户重建目录结构的数字资产管理系统。

## 2.3 产品竞争方式

SceneWeaver 不与 Premiere、Final Cut、DaVinci Resolve 正面竞争剪辑能力。

它通过以下方式建立差异：

1. 跨项目、跨文件夹、跨剪辑软件的统一素材索引；
2. 中文优先，同时支持英文和日文查询；
3. 支持 Must / Should / Must Not 的组合搜索；
4. 支持参考图、角色、人物、产品或视觉主体的用户自定义实体；
5. 通过场景包提供垂直行业理解；
6. 本地优先、模型可替换、数据可迁移；
7. 直接打开源文件、复制时间码、拖入剪辑软件或导出工程交换文件。

# 3. 目标用户与优先级

## 3.1 首发核心用户

### A. 重度个人视频创作者

特征：

- 拥有 100 GB 到 20 TB 素材；
- 每周或每月持续创作；
- 素材分散在多个文件夹和硬盘；
- 经常记得画面但忘记文件名和位置；
- 使用 Premiere、达芬奇、Final Cut、剪映专业版或 Blender。

核心痛点：找素材和选片耗时，而不是不会剪辑。

### B. ACG / 游戏 / 动漫二创作者

特征：

- 制作 AMV、MAD、GMV、角色混剪、剧情向视频；
- 需要按角色、表情、服装、战斗状态、游戏 UI、字幕、场景和镜头语言找片段；
- 中文和日文角色名、别名、作品名并存；
- 常使用角色参考图。

这是首个重点垂直场景，但不是产品的全部市场。

### C. 摄影、Cosplay 与短视频工作室

特征：

- 同时管理图片和视频；
- 需要找人物、服装、动作、机位、布景、产品或拍摄日期；
- 需要把结果快速交给剪辑师或客户。

## 3.2 第二阶段用户

- Vlog 与旅行视频创作者；
- 电商和产品视频团队；
- 访谈、播客和课程视频制作者；
- 游戏主播和赛事内容团队；
- 小型品牌内容团队；
- 研究、新闻和档案素材整理者。

## 3.3 暂不服务

- 只管理几十张照片的轻度用户；
- 需要云端多人审片和严格权限的大型影视公司；
- 需要广播级媒体资产管理的机构；
- 期待一句话生成完整成片的用户；
- 需要自动获取受版权保护素材的用户。

# 4. 用户任务 JTBD

## 4.1 通用任务

- 当我记得画面但不知道文件名时，我要快速找到它；
- 当我有几小时录屏时，我要直接跳到特定动作发生的时间；
- 当我想筛掉字幕、模糊、黑帧或 UI 时，我要一次排除；
- 当我找到多个候选时，我要快速预览、比较和收藏；
- 当我开始剪辑时，我要把片段和时间码送到原有软件；
- 当素材文件夹新增文件时，我不想重新扫描全部内容。

## 4.2 ACG 专属任务

- 找某个角色在某种场景、动作或表情下的镜头；
- 排除战斗、游戏 UI、字幕和正脸；
- 搜索角色服装形态、发色、视角和情绪；
- 找适合开头、高潮、转折或结尾的候选镜头；
- 用参考图定义一个模型不认识的新角色。

## 4.3 访谈和课程任务

- 搜索某句话或同义表达出现的位置；
- 搜索某位说话者；
- 找没有口误、画面稳定且声音清晰的片段；
- 汇总某个主题的所有相关片段。

# 6. 第一版核心闭环

## 6.1 首次启动

用户看到三步引导：

1. 选择素材文件夹；
2. 选择索引模式；
3. 开始后台扫描。

首次运行不得要求用户理解：

- Docker；
- 数据库；
- Python；
- Node.js；
- 模型名称；
- 向量维度；
- API Endpoint。

高级用户可以在设置中选择模型和 Provider。

## 6.2 索引模式

### 快速模式

适合首次体验或普通电脑：

- 视频均匀抽帧；
- 基础镜头切分；
- 小型本地视觉模型；
- 不进行 OCR 和语音转录；
- 优先快速获得可搜索结果。

### 平衡模式

默认模式：

- 自适应镜头切分；
- 每镜头 1～2 个代表帧；
- 视觉向量；
- 质量检测；
- 可选轻量 OCR；
- 后台低优先级运行。

### 精确模式

用户主动选择：

- 更多关键帧；
- 运动和时间变化分析；
- OCR；
- 语音转录；
- 结构化场景标签；
- 可选云端重排；
- 显示预计时间、磁盘占用和云端请求成本。

## 6.3 搜索

用户可以直接输入：

```text
雨夜里有人撑伞回头
红色跑车从左向右经过
访谈中提到“用户留存”的位置
粉色头发角色微笑，不要战斗和字幕
和这张参考图里同一个人物，优先近景
2025 年 8 月在室外拍摄的产品特写
```

系统解析并展示查询条件芯片：

```text
必须：人物、撑伞、回头
偏好：雨夜、冷色
排除：字幕、游戏 UI
```

用户可以直接删除或修改芯片，不需要重写整句查询。

## 6.4 结果页

每个结果展示：

- 缩略图或动态悬停预览；
- 文件名和所在素材库；
- 视频开始与结束时间；
- 匹配分数；
- 匹配原因；
- 未满足条件；
- 画质和字幕提示；
- 收藏、评分、备注；
- 打开原文件；
- 打开所在文件夹；
- 复制路径；
- 复制时间码；
- 加入选片集合。

## 6.5 选片集合 Selects

选片集合不是时间线。

它负责：

- 收藏候选素材；
- 分组；
- 排序；
- 评分；
- 添加备注；
- 标记推荐区间；
- 批量导出；
- 生成联系表或审片清单；
- 输出到剪辑软件。

第一版不承担复杂剪辑职责。

# 8. 搜索系统设计

## 8.1 搜索信号

搜索不得只依赖一个向量。

可用信号：

- 视觉向量；
- 参考图向量；
- 文件名；
- 文件路径；
- EXIF 和媒体元数据；
- OCR 文本；
- ASR 转录；
- 用户标签；
- 场景包标签；
- 质量分；
- 时间和日期；
- 收藏与用户反馈；
- 实体匹配；
- 镜头持续时间；
- 相邻镜头上下文。

## 8.2 查询结构

```python
class QuerySpec(BaseModel):
    raw_query: str
    must: list[Condition] = []
    should: list[Condition] = []
    must_not: list[Condition] = []
    text_terms: list[str] = []
    reference_asset_ids: list[UUID] = []
    entity_ids: list[UUID] = []
    library_ids: list[UUID] = []
    date_range: DateRange | None = None
    preferred_duration_ms: Range | None = None
    min_quality_score: float | None = None
    media_types: list[Literal["image", "video", "audio"]] = []
```

## 8.3 检索管线

### 第 1 层：确定性过滤

先处理：

- 素材库；
- 文件类型；
- 日期；
- 路径；
- 用户标签；
- 是否字幕；
- 是否 UI；
- 是否模糊；
- 时长；
- 明确排除条件。

### 第 2 层：多路召回

并行召回：

- 视觉文本向量；
- 参考图相似度；
- 全文检索；
- OCR；
- ASR；
- 实体相似度；
- 文件名和路径模糊匹配。

### 第 3 层：融合排序

综合：

- 语义相关度；
- 条件命中；
- 负条件违反；
- 质量分；
- 重复惩罚；
- 用户历史偏好；
- 结果多样性；
- 镜头长度适配。

### 第 4 层：可选精确重排

仅对前 20～50 个候选使用：

- 更强本地模型；
- 云端视觉模型；
- 场景包重排器。

云端模式必须是用户主动开启，并在调用前显示：

- 上传的关键帧数量；
- 总大小；
- 使用模型；
- 预计请求次数；
- 预计费用；
- 是否包含 OCR 或转录文本。

## 8.4 结果多样性

默认避免 Top 20 全部来自同一个连续镜头。

排序应加入：

- 相邻镜头去重；
- 同源视频多样性；
- 视觉近重复惩罚；
- 同一场景最大结果数；
- 用户可切换“最相关”与“更多样”。

## 8.5 反馈闭环

用户可以点击：

- 很符合；
- 部分符合；
- 不符合；
- 这是该实体；
- 不是该实体；
- 隐藏相似结果。

反馈用于：

- 调整本地标签；
- 更新实体样本；
- 训练轻量个性化排序参数；
- 改进场景包词典；
- 不默认上传到服务器。

# 9. 索引策略与性能

## 9.1 增量优先

每个文件记录：

- 规范化路径；
- 文件大小；
- 修改时间；
- 快速指纹；
- 可选完整哈希；
- 索引版本；
- 模型版本；
- 代理版本。

文件未变化时不得重复分析。

## 9.2 分层分析

### Level 0：文件发现

- 路径；
- 格式；
- 大小；
- 修改时间。

### Level 1：媒体探测

- 时长；
- 分辨率；
- 帧率；
- 编码；
- EXIF；
- 音轨；
- 字幕轨。

### Level 2：快速视觉索引

- 图片缩略图；
- 视频均匀抽帧或代表帧；
- 视觉向量；
- 基础质量检测。

完成后即可搜索，不等待后续任务。

### Level 3：镜头级索引

- 镜头检测；
- 关键帧；
- 短预览；
- 镜头向量；
- 相邻上下文。

### Level 4：增强理解

- OCR；
- ASR；
- 场景包标签；
- 实体匹配；
- 高精度重排缓存。

用户必须能在 Level 2 完成后立即使用，不得因为增强分析尚未完成而阻塞产品。

## 9.3 资源控制

必须支持：

- CPU 线程上限；
- GPU 开关；
- 后台低优先级；
- 仅空闲时运行；
- 暂停和恢复；
- 笔记本电池模式；
- 缓存容量上限；
- 每素材库独立配置；
- 任务崩溃后断点恢复。

## 9.4 缓存策略

缓存包括：

- 缩略图；
- 低清代理；
- 关键帧；
- 波形；
- OCR；
- ASR；
- 向量；
- 重排结果。

用户可以分别查看和清理，默认绝不删除原文件。

# 10. 技术架构重构

## 10.1 消费者版本目标

- Windows 一键安装；
- 首次启动不依赖 Docker；
- 不要求安装 Python 或 Node.js；
- 可离线完成基础索引和搜索；
- 无 API Key 也能使用核心功能；
- 数据目录可迁移和备份；
- 崩溃后任务可恢复。

## 10.2 推荐架构

```text
Tauri Desktop Shell
        │
React + TypeScript UI
        │ local IPC / localhost API
Native Supervisor
        │
Packaged Media & ML Worker
        ├─ FFmpeg / ffprobe
        ├─ Scene Detection
        ├─ Thumbnail / Proxy
        ├─ Local Embedding Runtime
        ├─ Optional OCR / ASR
        └─ Optional Cloud Provider

Storage
        ├─ SQLite: metadata, jobs, tags, entities, collections
        ├─ Embedded vector store: visual/text vectors
        ├─ FTS index: filename, OCR, transcript, notes
        └─ Local cache directory
```

## 10.3 数据存储决策

消费者版：

- SQLite 作为业务元数据数据库；
- WAL 模式；
- FTS5 用于文本检索；
- 嵌入式向量库使用 LanceDB，或在性能验证通过后采用稳定的 SQLite 向量扩展；
- 不运行独立数据库服务。

团队或服务器版未来可选：

- PostgreSQL；
- pgvector；
- 对象存储；
- 任务队列；
- 权限和协作。

消费者版本与团队版本共享领域模型，但不共享部署复杂度。

## 10.4 模型 Provider

所有模型通过统一接口：

```python
class VisualEmbeddingProvider(Protocol): ...
class TextEmbeddingProvider(Protocol): ...
class RerankProvider(Protocol): ...
class OCRProvider(Protocol): ...
class ASRProvider(Protocol): ...
class EntityMatchProvider(Protocol): ...
```

支持：

- 本地内置模型；
- 用户下载的模型；
- OpenAI-compatible API；
- 专业视频理解 API；
- Mock Provider。

业务代码不得直接依赖单一厂商 SDK。

## 10.5 模型选择原则

默认模型必须：

- 可在普通 Windows 电脑 CPU 上运行；
- 支持中文；
- 模型体积可接受；
- 许可证适合分发；
- 支持离线；
- 有明确版本锁定；
- 可被更高质量模型替换。

不要在产品规范中永久写死具体模型。每次发布前单独进行：

- 许可证审查；
- 中文检索测试；
- 动画与真人素材测试；
- CPU / GPU 性能测试；
- 模型下载体验测试。

## 10.6 本地与云端混合策略

默认：本地。

云端仅用于：

- 用户主动选择的精确重排；
- 少量候选的结构化描述；
- 本地模型不擅长的复杂时序判断；
- 用户明确授权的批处理。

禁止默认把整段原视频上传到云端。

- 任务可暂停；
- 任务可恢复；
- 单个损坏文件不阻塞整个素材库；
- 外接硬盘断开后不崩溃；
- 应用重启后队列恢复；
- 日志提供人类可读错误；
- 错误页面提供下一步操作。

## 14.5 无障碍与效率

- 完整键盘操作；
- 可调整缩略图大小；
- 支持深色与浅色；
- 支持中文、英文，后续日文；

- 支持快捷键收藏和打开；

## 阶段 7：助手与智能选片建议

只有前面核心指标达标后才允许开始。

助手可以：

- 根据目标描述创建搜索集合；
- 推荐查询改写；
- 从选片集合中建议结构；
- 发现缺少的镜头类型；

助手不得：

- 未经确认覆盖用户选择；
- 直接生成不可解释的最终成片；
- 引用素材库外内容；

# 21. 原规格迁移映射

- 视频导入 → 素材库导入；
- scenes → segments；
- jobs → 持久化可恢复任务；
- characters → 通用 entities；

- Provider 抽象 → 保留；

- 混合搜索 → 保留并扩展；
- 测试和指标 → 保留并改为用户结果导向。

- 视觉模型重排；
- OCR；
- ASR；
- EDL / FCPXML；

# 22. 风险矩阵

| 风险 | 严重度 | 应对方式 |
|---|---:|---|
| 主流剪辑软件内置语义搜索 | 高 | 跨软件素材库、中文、组合条件、场景包和实体参考图 |
| 索引时间太长 | 高 | 分层索引、Level 2 即可搜索、暂停恢复、增量扫描 |
| 普通电脑性能不足 | 高 | 快速模式、CPU 友好模型、线程限制、延迟增强分析 |
| 搜索结果不准 | 高 | 多路召回、条件芯片、用户修正、精确重排、评估集 |
| ACG 用户过窄 | 中 | 通用核心 + ACG 首发包，而不是 ACG 硬编码 |
| 产品过于通用失去特色 | 高 | 首发只做好 General + ACG，两条真实工作流 |
| 安装复杂 | 高 | 一键桌面安装、内置运行时、无 Docker |
| 云端成本失控 | 高 | 默认本地、候选重排、成本预估、用户主动开启 |
| 版权争议 | 中 | 不抓取、不分发、只处理用户合法素材 |
| 重新实现剪辑器 | 高 | 选片不等于时间线，优先导出和插件 |
| 模型许可证变化 | 中 | Provider 抽象、发布前许可证审查、模型可替换 |
| 用户不愿改变工作流 | 高 | 打开源文件、拖拽、交换格式、剪辑软件插件 |

# 23. 产品完成定义

一个版本只有同时满足以下条件才算完成：

- 普通用户可无命令行安装；
- 核心功能有端到端测试；
- Windows 路径和外接硬盘已测试；
- 单个损坏文件不会阻塞任务；
- 任务可恢复；
- 缓存可查看和清理；
- 原文件不会被误删；
- 隐私说明与实际行为一致；
- 搜索质量有固定测试集；
- 性能有公开基准；
- README 有真实截图和 Demo；
- 无 API Key 可使用核心功能；
- 没有阻塞性 TODO；
- 至少 5 位真实用户完成核心任务；
- 至少一项用户结果指标达到版本目标。

# 26. 最终产品原则

1. 先让用户找到素材，再谈 AI 创作；
2. 搜索是核心，时间线不是；
3. 通用核心不等于没有垂直特色；
4. ACG 是首发能力包，不是系统硬编码；
5. 用户不应为了使用产品学习部署；
6. 索引必须增量、可暂停、可恢复；
7. 第一批结果必须尽快可用；
8. 本地优先，云端可选；
9. 每个 AI 结果必须可解释、可修正；
10. 不移动、不锁死、不误删用户原素材；
11. 与成熟剪辑软件协作，而不是重复制造剪辑器；
12. 通过真实用户保留和导出素材来判断价值，而不是通过功能数量判断完成。
